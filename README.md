# Splunk SOC Brute Force Investigation

This project is a hands-on SOC lab where I used Splunk to investigate brute force login activity.

I worked through the full investigation process by reviewing logs, identifying suspicious failed login patterns, building detections, creating dashboards, and documenting the incident timeline.

The goal of this project was to practice how a SOC analyst would detect, investigate, and report on suspicious authentication activity.

## Skills Practiced

| Skill         | How I Used It                                |
| ------------- | -------------------------------------------- |
| Splunk        | Searched and reviewed security logs          |
| Log Analysis  | Identified failed login patterns             |
| Detection     | Built searches to find brute force behavior  |
| Investigation | Followed the activity from alert to timeline |
| Documentation | Created notes, findings, and a SOC playbook  |

## Project Outcome

This lab helped me practice real SOC analyst work by turning raw log data into an investigation. I used Splunk to find suspicious login behavior, explain what happened, and document the response.
